# Support

## Community Support

Use this repository's GitHub issues for reproducible bugs, documentation gaps,
and feature requests. During the public repository cutover, use whichever
repository is currently accepting Palace of Truth changes.

Before opening an issue, include:

- Palace version or commit SHA
- deployment mode, such as local Compose, Helm, or host-side dev
- relevant logs with secrets and user content removed
- reproduction steps and expected behavior

## Security Issues

Do not open public issues for vulnerabilities or sensitive operational details.
Use the private vulnerability reporting path described in [SECURITY.md](SECURITY.md).

## Deployment Notes

Do not put private registry credentials, cluster names, DNS targets, secret-manager item IDs, or deployment runbooks in public issues. External operators should replace maintainer examples with their own infrastructure coordinates.
